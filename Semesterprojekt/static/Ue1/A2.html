Aufgabe 1.2 - HTTP

Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?

  • "OK" - Der Request wurde bearbeitet und man bekommt das Ergebnis der Anfrage in der Response

Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?

  • "Moved Permanently"
  • Die Adresse war veraltet und man bekommt die neue Adresse im Location-Header-Feld überliefert (Redirect)

Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?

  • "Bad Request"
  • Es gab einen Fehler im Request. Als Endnutzer ohne weiteres Wissen kann man wenig tun, als z.B. mit Suchmaschinen nach einer Alternative schauen. Als Entwickler kann man in seinem Request auf Fehlersuche gehen, z.B. mit den Dev-Tools im Browser oder über die Konsole.

Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?

  • "Forbidden"
  • Als Client bin ich nicht berechtigt, auf die in meinem Request angefragte Ressource zuzugreifen. Dies kann z.B. daran liegen, dass man erst eine Authentifizierung z.B. in Form eines Logins durchlaufen muss.

In einer Webanwendung benötigen Sie eine OPTIONS-Anfrage, die die Optionen des Servers vor dem eigentlichen Zugriff erfragen soll. Aus Performanzgründen soll die Abfrage jedoch cacheable sein. Wie könnten Sie dafür eine Lösung angehen?

  • Die "Optionen" des Servers, also z.B. technische Daten, als Ressourcen hinterlegen und dann mit GET abfragen (und evtl. durch Authorisierung schützen)
